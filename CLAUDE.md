## Project Overview

TraceNet is a graph algorithms DAA (Design and Analysis of Algorithms) course project — B.E. ISE Semester IV, RVCE Bengaluru. It models antibiotic resistance gene (ARG) spread through bacterial transmission networks as a weighted directed graph, then applies eight classical graph algorithms across DAA Units II–V to analyze spread, community structure, path risk, and containment.

**Status:** The codebase is scaffolded — `.cpp`, `.h`, and `.py` files exist with documented interfaces but no implementation yet.

---

## Architecture

TraceNet has two independent layers connected only by the adjacency file format:

1. **Python preprocessing layer** (`preprocessing/build_graph.py`) — downloads CARD-R prevalence TSVs, filters species/ARGs, computes Jaccard-based weights, and emits `data/hgt_graph.txt`.
2. **C++ algorithm engine** (`src/`) — parses `hgt_graph.txt` and runs all eight algorithms. One `.cpp`/`.h` pair per algorithm module.

### Two distinct graph types — do not conflate

| Graph | Nodes | Edges | Used for |
|---|---|---|---|
| **HGT species graph** `G` | Bacterial species (20–25) | HGT pathways, directed + weighted | BFS, SCC, Dijkstra, Floyd-Warshall, containment |
| **ARG dependency DAG** `D` | Resistance gene families (10) | Co-occurrence dependencies, directed acyclic | Topological sort only |

Topological sort operates on `D` (loaded from `data/arg_dag.txt`), never on `G`.

### C++ source files — one file per algorithm

```
src/
├── main.cpp                  # CLI parsing, algorithm dispatch, graph loading
├── graph.h / graph.cpp       # Graph struct, adjacency list + matrix, I/O
├── bfs.h / bfs.cpp
├── scc_kosaraju.h / scc_kosaraju.cpp
├── topo_sort.h / topo_sort.cpp
├── boyer_moore.h / boyer_moore.cpp
├── dijkstra.h / dijkstra.cpp
├── floyd_warshall.h / floyd_warshall.cpp
├── greedy_contain.h / greedy_contain.cpp

```

## Data Sources and Pipeline

### Primary data sources
- **CARD-R Prevalence**: `card.mcmaster.ca/latest/variants` — per-species TSV files with ARG prevalence stats. Filter: `Contig == "Plasmid"` AND `Prevalence > 0.05`. Store in `data/card_r/` (gitignored, ~200MB).
- **Ellabaan 2021 Supplementary** (fallback/directionality): Nature Comms DOI `10.1038/s41467-021-22757-1` — provides explicit HGT directionality for ambiguous edges.
- **CARD FASTA** (for Boyer-Moore): `card.mcmaster.ca/latest/data` — `nucleotide_fasta_protein_homolog_model.fasta`. Pre-filtered to 6 ARGs (NDM-1, CTX-M-15, mcr-1, vanA, tetM, blaTEM-1), stored as `data/arg_sequences.fasta`.

### Python preprocessing — `preprocessing/build_graph.py`

Key constants:
```python
JACCARD_THRESHOLD = 0.10      # minimum ARG set overlap for an edge to exist
MIN_WEIGHT = 0.05             # minimum taxonomic-adjusted weight after penalty
PREVALENCE_CUTOFF = 0.05      # ARG must appear in >5% of genomes
```

Pipeline steps:
1. Load per-species ARG sets from CARD-R TSVs.
2. Compute pairwise Jaccard similarity: `J(A,B) = |ARGs(A) ∩ ARGs(B)| / |ARGs(A) ∪ ARGs(B)|`
3. Apply **taxonomic correction factor** `τ`:
   - Same genus: `τ = 1.0`
   - Same gram class, different genus: `τ = 0.75`
   - Cross gram-stain (Gram+ ↔ Gram−): `τ = 0.5`
4. Adjusted weight: `w(A→B) = J(A,B) × τ(A,B)` — skip edge if below thresholds.
5. Write `data/hgt_graph.txt`.

### Key data files

| File | Generated by | Description |
|---|---|---|
| `data/hgt_graph.txt` | `build_graph.py` | Main graph input for C++ engine |
| `data/arg_dag.txt` | Hand-authored | ARG dependency DAG for topological sort |
| `data/arg_sequences.fasta` | `download_fasta.py` | 6 reference ARG sequences for Boyer-Moore |
| `data/hospital_subgraph.txt` | Generated | Reduced ~10-node subgraph for B&B |
| `data/card_r/` | Downloaded | Raw CARD-R TSVs — **gitignored** |

## Graph Model

### HGT graph file format — `hgt_graph.txt`
```
<n_nodes>
<n_edges>
<node_0_name>
<node_1_name>
...
<src_idx> <tgt_idx> <weight> <ARG_label_1,ARG_label_2,...>
```
- `u`, `v` are zero-based integer node indices
- `weight` is the raw probability `w ∈ (0,1]` (Jaccard × τ), **not** the log-distance
- Labels are a comma-separated list of shared ARG family names (up to 5)

### Edge weight and Dijkstra distance

The file stores **probabilities**. Dijkstra converts internally:
```
distance(u→v) = -log(w(u→v))
```
High probability → low distance → preferred path. This transformation happens inside `dijkstra.cpp`, not in preprocessing. Floyd-Warshall uses the same -log(w) transformation.

### Expected graph properties
- `|V|` = 20–25 nodes, `|E|` = 80–180 edges
- Diameter ~4–6 hops from environmental source to clinical target
- Multiple SCCs of size 2–5; most singleton SCCs (recipients only)

### Species in the graph

| Role | Species examples |
|---|---|
| ESKAPE clinical targets | *K. pneumoniae*, *A. baumannii*, *P. aeruginosa*, *E. cloacae*, *E. faecium*, *S. aureus* |
| Extended clinical | *E. coli*, *S. enterica*, *C. difficile*, *S. pneumoniae* |
| Livestock reservoirs (source) | *E. coli* (livestock variant), *E. faecalis*, *C. jejuni* |
| Wastewater bridges | *A. pittii*, *P. putida* |
| Soil reservoirs (source) | *B. subtilis*, *S. coelicolor*, *Actinobacter sp.* |

### ARG dependency DAG — `data/arg_dag.txt`

10 ARG nodes, 8 hand-curated dependency edges:
```
tetM     → blaTEM        sul1     → blaTEM
blaTEM   → blaSHV        blaSHV   → blaCTX-M
blaCTX-M → blaOXA-48    blaOXA-48 → blaNDM-1
aac(6')-Ib → blaNDM-1   blaCTX-M → mcr-1
```
Expected topological order: `tetM → sul1 → blaTEM → blaSHV → aac(6')-Ib → blaCTX-M → blaOXA-48 → blaNDM-1 → mcr-1`

## Algorithm Design

### Algorithm responsibilities and key implementation choices

| Algorithm | File | Key choice | Purpose |
|---|---|---|---|
| BFS | `bfs.cpp` | Standard queue | Hop-distance reachability from source |
| SCC | `scc_kosaraju.cpp` | **Kosaraju** (not Tarjan) — two-pass DFS, pedagogically clearer | Resistance bubble detection |
| Topological Sort | `topo_sort.cpp` | **Kahn's algorithm** (BFS in-degree) — explicit cycle detection | ARG acquisition ordering on DAG `D` |
| Boyer-Moore | `boyer_moore.cpp` | Bad-character table, DNA 4-char alphabet | ARG sequence search in CARD FASTA |
| Dijkstra | `dijkstra.cpp` | Min-heap priority queue; convert w → -log(w) internally | Highest-probability spread path |
| Floyd-Warshall | `floyd_warshall.cpp` | Adjacency **matrix** (not list); INF for unreachable | All-pairs vulnerability matrix |
| Greedy containment | `greedy_contain.cpp` | Sort edges by weight descending; BFS reachability check after each removal | Approximate min-cut |
| Branch-and-Bound | `bnb_contain.cpp` | Run on `hospital_subgraph.txt` only (~10 nodes); prune when `removed.size() >= best_cost` | Exact containment, compare vs greedy |

### Data structure split

- Floyd-Warshall: needs O(V²) adjacency **matrix** internally.
- All other algorithms: adjacency **lists**.
- `graph.h` must support both representations (or build matrix lazily from list).

### Complexity reference

| Algorithm | Time | Space | Syllabus Unit |
|---|---|---|---|
| BFS | O(V + E) | O(V) | Unit II |
| SCC (Kosaraju) | O(V + E) | O(V + E) | Unit II |
| Topological Sort (Kahn's) | O(V + E) | O(V) | Unit II |
| Boyer-Moore | O(nm) worst / O(n/m) avg | O(4) for DNA | Unit III |
| Dijkstra | O((V + E) log V) | O(V) | Unit IV |
| Floyd-Warshall | O(V³) | O(V²) | Unit IV |
| Greedy min-cut | O(E log E + k(V+E)) | O(V+E) | Unit IV |
| Branch-and-Bound | O(2^E) worst / fast in practice | O(E) | Unit V |

With n=22, e≈147: Floyd-Warshall runs 22³ = 10,648 ops. All algorithms complete in <1 second.

### NP-hardness framing

- **Single-source s-t min-cut** (used in B&B): polynomial — solvable by max-flow. B&B is used as a pedagogical substitute since it's on the syllabus.
- **Multi-source containment** (full graph, multiple environmental sources → single clinical target): NP-hard by reduction from Minimum Vertex Cover.
- This justifies why the greedy approximation exists for the full graph and B&B is restricted to the small hospital subgraph.

## Key Design Constraints

- Python and C++ layers are fully independent — the only coupling is the adjacency file format (`hgt_graph.txt`).
- `main.cpp` owns CLI argument parsing and algorithm dispatch; no algorithm logic lives there.
- Edge weights in the file are raw probabilities `w ∈ (0,1]`. The `-log(w)` distance conversion happens **inside `dijkstra.cpp` and `floyd_warshall.cpp`**, not in preprocessing.
- Prefer clarity over micro-optimization — every algorithm must be explainable step-by-step in a viva.
- B&B runs only on `hospital_subgraph.txt` (~10 nodes). Never attempt B&B on the full 22-node graph.
- The ARG dependency DAG has no cycles by construction. Kahn's algorithm should assert this — if output size < `|A|`, a cycle exists and the DAG input is invalid.

## Modeling Assumptions

1. Edges are analytical abstractions — they do not claim observed HGT in every case.
2. Edge weights (Jaccard × τ) are structured proxies, not verified biological transfer probabilities.
3. The ARG dependency DAG models clinical co-occurrence patterns, not strict chemical prerequisites.
4. "Blocking an edge" in containment is an abstract graph operation, not a literal biological intervention.
5. Boyer-Moore runs on 300–2000 bp reference sequences from CARD, not full genomic data.
6. This is not a medical or clinical tool.
