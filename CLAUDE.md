## Project Overview

TraceNet is a graph algorithms DAA (Design and Analysis of Algorithms) course project вЂ” B.E. ISE Semester IV, RVCE Bengaluru. It models antibiotic resistance gene (ARG) spread through bacterial transmission networks as a weighted directed graph, then applies eight classical graph algorithms across DAA Units IIвЂ“V to analyze spread, community structure, path risk, and containment.

**Status:** The codebase is scaffolded вЂ” `.cpp`, `.h`, and `.py` files exist with documented interfaces but no implementation yet.

## Architecture

TraceNet has two independent layers connected only by the adjacency file format:

1. **Python preprocessing layer** (`preprocessing/build_graph.py`) вЂ” downloads CARD-R prevalence TSVs, filters species/ARGs, computes Jaccard-based weights, and emits `data/hgt_graph.txt`.
2. **C++ algorithm engine** (`src/`) вЂ” parses `hgt_graph.txt` and runs all eight algorithms. One `.cpp`/`.h` pair per algorithm module.

### Two distinct graph types вЂ” do not conflate

| Graph | Nodes | Edges | Used for |
|---|---|---|---|
| **HGT species graph** `G` | Bacterial species (20вЂ“25) | HGT pathways, directed + weighted | BFS, SCC, Dijkstra, Floyd-Warshall, containment |
| **ARG dependency DAG** `D` | Resistance gene families (10) | Co-occurrence dependencies, directed acyclic | Topological sort only |

Topological sort operates on `D` (loaded from `data/arg_dag.txt`), never on `G`.

## Data Sources and Pipeline

> Full dataset documentation with verified schemas, filter rationale, complete Jaccard matrix, and ARG name mapping is in **`docs/dataset_reference.md`**. That file is the authoritative source. This section summarises the key decisions only.

### Downloaded datasets (already on disk)
- **`data/card_r/card_prevalence.txt.gz`** вЂ” CARD-R Prevalence (CARD v4.0.2). 20,041 rows Г— 11 columns. Primary source for graph construction.
- **`data/card_fasta/nucleotide_fasta_protein_homolog_model.fasta`** вЂ” CARD Reference FASTA (CARD v4.0.1). 6,052 sequences, median 873 bp. Used for Boyer-Moore.

### The correct filter for `build_graph.py`

```python
# Load HGT-relevant (mobile, plasmid-borne) ARGs for one species
sub = df[
    (df["Pathogen"] == species_name) &
    (df["NCBI Plasmid"] > 1) &            # >1% of genomes carry it on a plasmid
    (df["Model Type"] == "protein homolog model")
]
arg_set = set(sub["Name"].unique())
```

**Do NOT use `NCBI WGS >= 5`** вЂ” that filter picks up intrinsic chromosomal genes (`rsmA`, `CRP`, `H-NS`, `ArnT`) that are never horizontally transferred and would pollute the Jaccard scores.

### Actual column names in `card_prevalence.txt.gz`

The spec's assumed column names are **wrong**. Real names verified by `head`:

| Spec assumed | Actual column | Semantics |
|---|---|---|
| `"Contig"` | does not exist | вЂ” |
| `"ARO_name"` | `"Name"` | allele-level gene name (e.g., `TEM-1`, `NDM-6`) |
| `"Prevalence"` (float 0вЂ“1) | `"NCBI Plasmid"`, `"NCBI WGS"` | percentage 0вЂ“100, not a proportion |
| вЂ” | `"Criteria"` | `"perfect"` or `"perfect_strict"` only вЂ” keep both |

### Python preprocessing вЂ” `preprocessing/build_graph.py`

Key constants (updated from EDA):
```python
JACCARD_THRESHOLD = 0.10      # minimum Jaccard before П„ correction
MIN_WEIGHT        = 0.05      # minimum w = Jaccard Г— П„
PLASMID_CUTOFF    = 1.0       # NCBI Plasmid must exceed this (percentage, 0вЂ“100 scale)
MODEL_TYPE        = "protein homolog model"
```

Pipeline steps:
1. Load `data/card_r/card_prevalence.txt.gz` with `pd.read_csv(..., sep="\t", compression="gzip")`.
2. For each species, filter rows: `NCBI Plasmid > 1` AND `Model Type == "protein homolog model"`. Build `arg_set = set(df["Name"])`.
3. Compute pairwise Jaccard: `J(A,B) = |ARGs(A) в€© ARGs(B)| / |ARGs(A) в€Є ARGs(B)|`
4. Apply taxonomic correction `П„` (same genus=1.0, same Gram class=0.75, cross Gram=0.5).
5. Skip edge if `J < 0.10` or `J Г— П„ < 0.05`.
6. Write `data/hgt_graph.txt` (bidirectional edges, equal weight both directions).

## Graph Model

### HGT graph file format вЂ” `hgt_graph.txt`
```
<n_nodes>
<n_edges>
<node_0_name>
<node_1_name>
...
<src_idx> <tgt_idx> <weight> <ARG_label_1,ARG_label_2,...>
```
- `u`, `v` are zero-based integer node indices
- `weight` is the raw probability `w в€€ (0,1]` (Jaccard Г— П„), **not** the log-distance
- Labels are a comma-separated list of shared ARG family names (up to 5)

### Edge weight and Dijkstra distance

The file stores **probabilities**. Dijkstra converts internally:
```
distance(uв†’v) = -log(w(uв†’v))
```
High probability в†’ low distance в†’ preferred path. This transformation happens inside `dijkstra.cpp`, not in preprocessing. Floyd-Warshall uses the same -log(w) transformation.

### Confirmed graph properties (from EDA + pipeline run)
- `|V|` = **16 nodes** (confirmed вЂ” 4 originally planned species dropped, see below)
- `|E|` = **144 directed edges** (72 undirected pairs Г— 2, all bidirectional with equal weight)
- Jaccard range across edges: 0.10 вЂ“ 0.71; w (Jaccard Г— П„) range: 0.050 вЂ“ 0.714
- Highest-weight pair: *E. faecium* в†” *E. faecalis* (Jaccard=0.714, П„=1.0 same genus, w=0.714)
- **Two connected components** (discovered by running the pipeline):
  - **Component A (12 nodes):** all Gram-negative species вЂ” K. pneumoniae, E. cloacae, P. aeruginosa, A. baumannii, E. coli, S. enterica, K. oxytoca, C. freundii, P. mirabilis, S. marcescens, A. pittii, P. putida
  - **Component B (4 nodes):** E. faecium, S. aureus, E. faecalis, C. jejuni
  - Cross-Gram Jaccard Г— П„(0.5) values all fall below MIN_WEIGHT=0.05 вЂ” no edges exist between the components. This is biologically valid (ARG overlap across Gram boundaries is structurally insufficient at these thresholds).
  - Each component is internally a **complete directed graph** (all within-component pairs above threshold).
- Gram-positive species (П„=0.5 vs Gram-negatives): *E. faecium*, *E. faecalis*, *S. aureus*

## Algorithm Design

### Algorithm responsibilities and key implementation choices

| Algorithm | File | Key choice | Purpose |
|---|---|---|---|
| BFS | `bfs.cpp` | Standard queue | Hop-distance reachability from source |
| SCC | `scc_kosaraju.cpp` | **Kosaraju** (not Tarjan) вЂ” two-pass DFS, pedagogically clearer | Resistance bubble detection |
| Topological Sort | `topo_sort.cpp` | **Kahn's algorithm** (BFS in-degree) вЂ” explicit cycle detection | ARG acquisition ordering on DAG `D` |
| Boyer-Moore | `boyer_moore.cpp` | Bad-character table, DNA 4-char alphabet | ARG sequence search in CARD FASTA |
| Dijkstra | `dijkstra.cpp` | Min-heap priority queue; convert w в†’ -log(w) internally | Highest-probability spread path |
| Floyd-Warshall | `floyd_warshall.cpp` | Adjacency **matrix** (not list); INF for unreachable | All-pairs vulnerability matrix |
| Greedy containment | `greedy_contain.cpp` | Sort edges by weight descending; BFS reachability check after each removal | Approximate min-cut |
| Branch-and-Bound | `bnb_contain.cpp` | Run on `hospital_subgraph.txt` only (~10 nodes); prune when `removed.size() >= best_cost` | Exact containment, compare vs greedy |

### Data structure split

- Floyd-Warshall: needs O(VВІ) adjacency **matrix** internally.
- All other algorithms: adjacency **lists**.
- `graph.h` must support both representations (or build matrix lazily from list).

### Complexity reference

| Algorithm | Time | Space | Syllabus Unit |
|---|---|---|---|
| BFS | O(V + E) | O(V) | Unit II |
| SCC (Kosaraju) | O(V + E) | O(V + E) | Unit II |
| Topological Sort (Kahn's) | O(V + E) | O(V) | Unit II |
| Boyer-Moore | O(nm) worst / O(n/m) avg | O(4) for DNA | Unit III |
| Dijkstra | O((V + E) log V) | O(V) | Unit IV |
| Floyd-Warshall | O(VВі) | O(VВІ) | Unit IV |
| Greedy min-cut | O(E log E + k(V+E)) | O(V+E) | Unit IV |
| Branch-and-Bound | O(2^E) worst / fast in practice | O(E) | Unit V |

With n=22, eв‰€147: Floyd-Warshall runs 22Ві = 10,648 ops. All algorithms complete in <1 second.

### NP-hardness framing

- **Single-source s-t min-cut** (used in B&B): polynomial вЂ” solvable by max-flow. B&B is used as a pedagogical substitute since it's on the syllabus.
- **Multi-source containment** (full graph, multiple environmental sources в†’ single clinical target): NP-hard by reduction from Minimum Vertex Cover.
- This justifies why the greedy approximation exists for the full graph and B&B is restricted to the small hospital subgraph.

## Key Design Constraints

- Python and C++ layers are fully independent вЂ” the only coupling is the adjacency file format (`hgt_graph.txt`).
- `main.cpp` owns CLI argument parsing and algorithm dispatch; no algorithm logic lives there.
- Edge weights in the file are raw probabilities `w в€€ (0,1]`. The `-log(w)` distance conversion happens **inside `dijkstra.cpp` and `floyd_warshall.cpp`**, not in preprocessing.
- Prefer clarity over micro-optimization вЂ” every algorithm must be explainable step-by-step in a viva.
- B&B runs only on `hospital_subgraph.txt` (~10 nodes). Never attempt B&B on the full 16-node graph.
- The ARG dependency DAG has no cycles by construction. Kahn's algorithm should assert this вЂ” if output size < `|A|`, a cycle exists and the DAG input is invalid.
- The CARD `Name` column uses allele-level names (`TEM-1`, `NDM-6`). For Jaccard overlap, group by family using `str.startswith("TEM-")` etc. See `docs/dataset_reference.md` В§9 for the complete mapping.
- To validate `build_graph.py` output: check generated edge weights against the pre-computed Jaccard matrix in `docs/dataset_reference.md` В§7. Weights should match within floating-point rounding.

## Modeling Assumptions

1. Edges are analytical abstractions вЂ” they do not claim observed HGT in every case.
2. Edge weights (Jaccard Г— П„) are structured proxies, not verified biological transfer probabilities.
3. The ARG dependency DAG models clinical co-occurrence patterns, not strict chemical prerequisites.
4. "Blocking an edge" in containment is an abstract graph operation, not a literal biological intervention.
5. Boyer-Moore runs on 300вЂ“2000 bp reference sequences from CARD, not full genomic data.
6. This is not a medical or clinical tool.
